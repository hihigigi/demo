# 예선이도 이해하는 Spring Boot 🚀

> 초보자를 위한 완벽 가이드

---

## 📖 Spring Boot가 뭐예요?

Spring Boot는 **자바(Java)로 웹 애플리케이션을 쉽게 만드는 프레임워크**입니다.

### 비유로 이해하기
- 🏗️ **프레임워크** = 집을 지을 때 필요한 틀
- 처음부터 벽돌을 하나하나 쌓는 게 아니라, 이미 만들어진 뼈대를 사용하는 거죠.

---

## 🌐 웹 애플리케이션이란?

우리가 인터넷에서 매일 사용하는 것들입니다:

- 🔍 네이버, 구글
- 🎬 유튜브
- 📱 인스타그램

**당신의 핸드폰 브라우저에서 사용하는 모든 것이 웹 애플리케이션입니다.**

---

## ⚙️ Spring Boot 없이는 어떻게 되나?

Spring Boot 없이 자바로 웹앱을 만들려면:

```
1️⃣ 웹 서버 설정        → 복잡함
2️⃣ 데이터베이스 연결   → 복잡함
3️⃣ 로그인 기능        → 복잡함
4️⃣ 보안 설정          → 매우 복잡함
```

이 모든 걸 직접 코드로 처리해야 하는데... **정말 복잡해요!** 😫

---

## ✨ Spring Boot가 해주는 일들

Spring Boot가 나타나면 모든 게 쉬워집니다:

| 필요한 것 | Spring Boot의 도움 |
|---------|------------------|
| 웹 서버 | ✅ 자동으로 준비됨 (Tomcat) |
| 데이터베이스 | ✅ 쉽게 연결 가능 |
| 로그인 기능 | ✅ 기본 코드 제공 |
| 보안 | ✅ 자동으로 보호됨 |

**마치 레고 블럭을 끼우듯이 필요한 기능을 붙이면 되는 거예요!** 🧱

## 🎯 우리 프로젝트: 게시판 애플리케이션

이 프로젝트는 다음 기능을 가진 **게시판 앱**입니다:

- ✍️ **회원가입** - 회원 정보 저장
- 🔐 **로그인** - 사용자 인증
- 📝 **게시글 작성** - 글 쓰기
- 👀 **게시글 조회** - 글 보기
- ✏️ **게시글 수정** - 글 수정하기
- 🗑️ **게시글 삭제** - 글 지우기

---

## 💻 주요 폴더 상세 설명

### [1] Controller (컨트롤러)

**역할:** 사용자의 요청을 받고 답변 보내기

```
사용자가 /login 페이지를 요청했다
  ↓
AuthController가 받음
  ↓
로그인 화면 보여주기
```

**주요 파일:**
- `AuthController.java` - 회원가입, 로그인
- `HomeController.java` - 홈 화면
- `PostController.java` - 게시글 관련 처리

---

### [2] Entity (엔티티)

**역할:** 데이터 모양을 정의하는 곳

**예시:**
```
사용자 = 이름 + 이메일 + 비밀번호
게시글 = 제목 + 내용 + 작성자
```

**주요 파일:**
- `User.java` - 회원 정보 모양 정의
- `Post.java` - 게시글 모양 정의

---

### [3] Repository (저장소)

**역할:** 데이터를 저장, 조회, 수정, 삭제

**예시:**
```
회원 정보를 데이터베이스에 저장하기
특정 사용자 찾기
```

**주요 파일:**
- `UserRepository.java` - 회원 관리
- `PostRepository.java` - 게시글 관리

---

### [4] Service (서비스)

**역할:** 실제 기능 처리 (비즈니스 로직)

**예시:**
```
새로운 회원가입 처리하기
게시글 목록 가져오기
```

**주요 파일:**
- `UserService.java` - 회원 기능
- `PostService.java` - 게시글 기능

---

### [5] Templates (템플릿)

**역할:** 사용자가 보는 화면 (HTML)

**주요 파일:**
- `login.html` - 로그인 화면
- `register.html` - 회원가입 화면
- `post-list.html` - 게시글 목록
- `post-write.html` - 글쓰기 화면

---

## 🔄 요청 처리 흐름 (실제 흐름)

```
사용자가 "게시글 쓰기" 버튼 클릭
    ↓
PostController가 요청 받음
    ↓
PostService에서 실제 처리
    ↓
PostRepository가 데이터베이스에 저장
    ↓
"글이 저장되었습니다!" 메시지
    ↓
post-list.html로 게시글 목록 화면 표시
```

## 📊 데이터베이스 (H2)

이 프로젝트는 **H2라는 간단한 데이터베이스**를 사용합니다.

### 왜 H2?
- ✅ 설치가 간단
- ✅ 테스트하기 좋음
- 📌 실제 프로젝트에서는 MySQL, PostgreSQL 등 사용

### 데이터는 엑셀처럼 테이블 형태로 저장됩니다

**users 테이블:**
```
┌────┬──────────┬──────────┬────────────────────┐
│ id │ username │ password │ email              │
├────┼──────────┼──────────┼────────────────────┤
│ 1  │ user1    │ 1234     │ user1@example.com  │
│ 2  │ user2    │ 5678     │ user2@example.com  │
└────┴──────────┴──────────┴────────────────────┘
```

**posts 테이블:**
```
┌────┬────────┬──────────┬──────────────────┬───────────┐
│ id │ title  │ content  │ author_id        │ created   │
├────┼────────┼──────────┼──────────────────┼───────────┤
│ 1  │ 첫글   │ 안녕하세요 │ 1               │ 2025-11-20│
│ 2  │ 두번글 │ 반갑습니다 │ 2               │ 2025-11-20│
└────┴────────┴──────────┴──────────────────┴───────────┘
```

## 🚀 이 프로젝트 실행하기

### 1단계: 빌드
```bash
./mvnw clean install
```
모든 코드를 컴파일하고 필요한 라이브러리를 다운로드합니다.

### 2단계: 실행
```bash
java -jar target/demo-0.0.1-SNAPSHOT.jar
```
빌드된 프로그램을 실행합니다.

### 3단계: 접속
웹 브라우저에서 접속:
```
http://localhost:8080
```

---

## ⚙️ 프로젝트 설정 (application.properties)

```properties
# 포트 설정
server.port=8080
→ 웹 서버가 8080번 포트에서 실행

# H2 콘솔 활성화
spring.h2.console.enabled=true
→ H2 데이터베이스 관리 페이지 활성화

# 데이터 저장 위치
spring.datasource.url=jdbc:h2:~/testdb
→ 데이터를 사용자 홈 폴더에 testdb라는 이름으로 저장
```

## 📝 Java 기본 개념

### [1] @Annotation (골뱅이 문법)

**역할:** Spring에게 "이 클래스는 이런 역할을 해!"라고 알려주기

```java
@Entity        → 데이터베이스 테이블이다
@Controller    → 사용자 요청을 처리한다
@Service       → 기능을 처리한다
@Repository    → 데이터를 다룬다
```

### [2] private, public (접근 제한)

```
private  → 같은 클래스에서만 사용 가능
public   → 어디서나 사용 가능
```

### [3] getter, setter (데이터 접근)

```
getter → 데이터를 읽기
setter → 데이터를 쓰기
```

💡 Lombok의 `@Getter`, `@Setter`가 자동으로 만들어줍니다.

### [4] 상속 (extends)

```
JpaRepository를 상속
  ↓
기본적인 데이터 저장 기능이 자동으로 생김
```

## 💡 왜 이렇게 나눠놨나? (MVC 패턴)

```
┌─────────────────────────────────────────┐
│         MVC 패턴                        │
├─────────────────────────────────────────┤
│ M (Model - Entity)                      │
│ - 데이터의 모양을 정의                  │
│ - 무엇을 저장할 것인가?                 │
├─────────────────────────────────────────┤
│ V (View - Templates)                    │
│ - 사용자가 보는 화면                    │
│ - HTML 파일들                           │
├─────────────────────────────────────────┤
│ C (Controller)                          │
│ - M과 V를 연결                          │
│ - 사용자 요청을 받아 적절히 처리        │
└─────────────────────────────────────────┘
```

### 이렇게 분리하면?
- ✅ 코드가 정리되고
- ✅ 버그를 찾기 쉽고
- ✅ 수정할 때 다른 부분 영향 적음

## 🐛 발생할 수 있는 문제들과 해결법

### "404 에러" - 페이지를 찾을 수 없다
```
원인: URL이 잘못되었거나, 컨트롤러에서 처리하지 않음
해결: URL과 매핑된 컨트롤러 메서드 확인
```

### "500 에러" - 서버 내부 오류
```
원인: 코드에 버그가 있음
해결: 콘솔에 에러 메시지 확인
```

### "데이터가 저장 안 됨"
```
확인사항:
1. Repository 메서드를 제대로 호출했는지 확인
2. 데이터베이스 연결이 제대로 되었는지 확인
```

### "화면이 안 뜬다"
```
확인사항:
1. templates 폴더에 HTML 파일이 있는지 확인
2. 컨트롤러에서 올바른 파일명을 반환하는지 확인
```
